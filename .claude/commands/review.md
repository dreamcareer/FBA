# コードレビュー

変更されたファイルをレビューし、以下の観点でフィードバックしてください:

1. **型安全性**: TypeScript の型が正しく使われているか
2. **APIレスポンス**: `ApiResponse<T>` 形式に従っているか
3. **エラーハンドリング**: try-catch が適切か、ステータスコードは正しいか
4. **セキュリティ**: SQLインジェクション、XSS、環境変数の漏洩がないか
5. **ビジネスロジック**: 在庫閾値や納品計算ルールが正しいか
6. **Prisma**: N+1問題がないか、include/select が適切か

```bash
git diff --staged
```

上記の差分をもとにレビューしてください。問題がなければ「LGTM」と回答してください。
